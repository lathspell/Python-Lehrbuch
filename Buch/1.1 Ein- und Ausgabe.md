# 1.1 Ein- und Ausgabe

Wie der Computer mit dir sprechen kann

---

## 🎯 Problem

Stell dir vor, du möchtest, dass der Computer dich nach deinem Namen fragt und dann "Hallo [Dein Name]" sagt. Oder du möchtest zwei Zahlen eingeben und der Computer soll dir die Summe anzeigen.

**Wie kann der Computer mit dir kommunizieren?**

---

## 📚 Theorie

Damit ein Programm mit dir "sprechen" kann, braucht es zwei einfache Befehle:

### Ausgabe: `print()`

Die `print()`-Funktion zeigt Text oder Ergebnisse auf dem Bildschirm an.

```python
print("Hallo Welt")  # Schreibt "Hallo Welt" auf den Bildschirm
```

### Eingabe: `input()`

Die `input()`-Funktion wartet, bis du etwas eingibst, und gibt dann deine Eingabe als Text zurück.

```python
name = input("Wie heißt du? ")  # Fragt nach deinem Namen
```

### Wie du mit dem Computer sprichst: Funktionen verstehen

Jetzt weißt du schon zwei Dinge, die der Computer für dich tun kann: `print()` und `input()`. Aber was sind das eigentlich?

**Funktionen sind die Worte, mit denen du dem Computer sagst, was er tun soll.**

- `print()` ist wie das Wort "Sag"
- `input()` ist wie das Wort "Frag"

Jede Funktion hat einen **Namen** (wie `print` oder `input`) und kann **Parameter** in ihren Klammern `()` entgegennehmen. Parameter sind die Informationen, die du der Funktion gibst, damit sie ihre Arbeit machen kann.

Beispiele:
- `print("Hallo")` – Die Funktion `print` bekommt den Parameter `"Hallo"` und zeigt diesen Text an
- `input("Name? ")` – Die Funktion `input` bekommt den Parameter `"Name? "` und zeigt diese Frage an, bevor sie auf deine Eingabe wartet

Manche Funktionen geben auch etwas zurück – das nennt man **Rückgabewert**.
- `input()` gibt deine Eingabe als Text zurück. Deshalb können wir schreiben: `name = input("Wie heißt du? ")` – der Rückgabewert (deine Eingabe) wird in der Variable `name` gespeichert
- `print()` gibt nichts zurück, sie zeigt nur etwas an

### Warum das mächtig ist

Der Computer ist nicht wählerisch! Ihm ist es egal, was du ihm als Parameter gibst, solange er damit etwas anfangen kann. Das bedeutet:

**Du kannst sogar Funktionen als Parameter für andere Funktionen benutzen!**

Stell dir vor, du möchtest sofort sehen, was jemand eingegeben hat. Du könntest so schreiben:

```python
print(input("Sag mir etwas: "))
```

Hier passiert Folgendes:
1. `input("Sag mir etwas: ")` wird zuerst ausgeführt und wartet auf deine Eingabe
2. Deine Eingabe (z.B. "Python ist toll") ist der Rückgabewert von `input()`
3. Dieser Rückgabewert wird direkt als Parameter an `print()` übergeben
4. `print()` zeigt dann deine Eingabe an

Das ist, als würdest du zum Computer sagen: "Frag den Benutzer nach etwas und sag dann sofort, was er gesagt hat."

Du siehst: Der Computer kann komplexe Befehle verstehen, wenn du sie aus einfachen Bausteinen (Funktionen) zusammenbaust. Und das Schöne ist – du wirst später lernen, wie du deine eigenen Funktionen erfindest!

---

## 📋 Zusammenfassung

| Konzept          | Beispiel                 | Beschreibung                                  |
|------------------|--------------------------|-----------------------------------------------|
| `print(texte)`   | `print("Hallo")`         | Zeigt Text auf dem Bildschirm an              |
| `input(frage)`   | `name = input("Name? ")` | Stellt eine Frage und gibt die Antwort zurück |
| **Funktion**     | `print()`, `input()`     | Ein Befehl, den der Computer ausführen kann   |
| **Parameter**    | `print("Hallo")`         | Information, die du der Funktion gibst        |
| **Rückgabewert** | `input(...)`             | Das Ergebnis, das eine Funktion zurückgibt    |

---

## ✅ Lösung

Hier ist ein einfaches Programm, das den Computer mit dir sprechen lässt:

```python
# Frag nach dem Namen
name = input("Wie heißt du? ")

# Begrüße die Person
print("Hallo, " + name + "! Schön, dich kennenzulernen.")
```

Wenn du dieses Programm ausführst, passiert Folgendes:

1. Der Computer fragt: "Wie heißt du?"
2. Du gibst deinen Namen ein (z.B. "Anna")
3. Der Computer antwortet: "Hallo, Anna! Schön, dich kennenzulernen."
